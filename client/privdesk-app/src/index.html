连接组件被安全软件拦截

PrivDesk 的连接功能被系统或安全软件拦截了。这是远程工具的常见误报(并非病毒),请按下面任一方法放行后重试:

Windows 智能应用控制
设置 → 隐私和安全性 → Windows 安全中心 → 应用和浏览器控制 → 智能应用控制 → 选择"关闭"
360 / 火绒 / 电脑管家
打开杀毒软件 → 找到拦截记录 / 信任区 → 把 PrivDesk 添加为"信任"或"允许运行"
Windows Defender
Windows 安全中心 → 病毒和威胁防护 → 管理设置 → 排除项 → 添加 PrivDesk 安装目录

放行后,回到 PrivDesk 重新点击"连接"即可。

macOS 拦截了 PrivDesk 的连接组件。这是未签名应用的常见情况(并非病毒),请按下面方法放行后重试:

首次打开被拦("已损坏 / 无法验证开发者")
在"访达 → 应用程序"里找到 PrivDesk,按住 Control 键点击图标 → 打开,在弹窗里再点"打开"即可。
仍打不开(提示"已损坏")
打开"终端",粘贴执行:
xattr -dr com.apple.quarantine /Applications/PrivDesk.app
然后重新打开。
系统设置里放行
系统设置 → 隐私与安全性 → 拉到底,看到 PrivDesk 被阻止 → 点"仍要打开"。

放行后,回到 PrivDesk 重新点击"连接"即可。
